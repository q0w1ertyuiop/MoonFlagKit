\section{项目简介}

MoonFlagKit 是一个用 MoonBit 实现的功能开关决策库。它接收功能开关配置和运行时上下文，返回某个功能是否应该开启，并给出简短原因。项目当前版本不做配置中心、控制台、网络同步或数据统计，只把核心判断逻辑先做稳定，方便后续被 Web 服务、Wasm 边缘函数、命令行工具或教学示例复用。

这个选题来自实际工程里常见的需求：新功能不能一次性放给所有用户，出问题时也需要快速关闭；有些功能只希望在生产环境、指定地区、指定用户或一小部分流量中开启。MoonFlagKit 把这些判断抽成一个小内核，让 MoonBit 项目可以先具备可测试的灰度发布能力。

\section{项目方向与适用场景}

项目方向属于基础工程组件，重点是运行时配置决策和渐进式发布。它适合以下场景：

\begin{itemize}
  \item MoonBit 服务端示例中按用户、环境或地区开启功能；
  \item Wasm 或边缘函数中根据本地配置做轻量判断；
  \item 命令行工具或自动化流程中控制实验性逻辑；
  \item 教学项目中展示功能开关、灰度发布、kill switch 等工程概念；
  \item 后续扩展为 OpenFeature 风格 SDK、远程配置 provider 或实验分流组件。
\end{itemize}

\section{技术路线}

项目把功能开关定义为 \texttt{Flag}，把运行时输入定义为 \texttt{EvalContext}。评估过程按固定顺序执行：先判断 flag 是否整体关闭，再检查 denylist，然后检查 allowlist，之后校验环境和属性条件，最后进入百分比灰度分桶。

灰度分桶使用 \texttt{bucket(subject, salt)} 得到 \texttt{0..99} 的稳定整数。只要用户标识和 salt 不变，结果就不会漂移。这样可以避免一个用户在多次请求中反复进入和退出同一功能。

解析器暂时采用行式配置，例如：

\begin{verbatim}
flag checkout_v2 enabled=true rollout=35 salt=summer
allow checkout_v2 ops-admin
deny checkout_v2 blocked-user
when checkout_v2 environment=prod
\end{verbatim}

这种格式不追求覆盖所有配置语言，而是服务于当前阶段的演示、测试和手写配置。后续可以在不改核心评估函数的情况下增加 JSON、TOML 或远程 provider。

\section{原创与不重合说明}

本项目为原创实现，不移植已有开源项目代码，不引入外部 MoonBit 依赖。项目重新选择了功能开关与灰度发布决策方向，源码、测试、文档和 CLI 示例均围绕这一方向重新组织，不复用之前仓库的业务设计。

在重新选题前，已检索 Mooncakes 上的相关项目，发现原方向已有同类包，因此改为 MoonFlagKit。功能开关与灰度发布决策在后端和云原生工程中比较成熟，同时目前在 MoonBit 包生态中不是常见主题，适合作为可继续扩展的基础组件。

\section{当前完成情况}

仓库包含 MoonBit 核心库、黑盒测试、CLI 示例、README、设计文档、验收清单和本申报材料。当前测试覆盖功能开关关闭、白名单、黑名单、属性条件、稳定分桶、配置解析和诊断输出。

后续计划包括：支持多变体实验、补充 JSON/TOML 配置入口、增加更丰富的条件表达式、提供 OpenFeature 风格接口，并尝试和 MoonBit Web/Wasm 示例项目集成。

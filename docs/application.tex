\textbf{项目名称：}MoonFlagKit，面向 MoonBit 的轻量级功能开关与灰度发布决策引擎。

\textbf{仓库链接：}
\begin{itemize}
  \item GitHub：{\small\url{https://github.com/q0w1ertyuiop/MoonFlagKit}}
  \item GitLink：{\small\url{https://www.gitlink.org.cn/a18265059561/MoonFlagKit}}
\end{itemize}

\textbf{项目简介：}
MoonFlagKit 是一个用 MoonBit 实现的功能开关决策库。它接收功能开关配置和运行时上下文，返回某个功能是否应该开启，并给出可读原因。项目当前版本不做配置中心、控制台、网络同步或统计系统，而是先把核心判断逻辑做稳定，方便被 MoonBit Web 服务、Wasm 边缘函数、命令行工具和教学示例复用。

\textbf{方向与价值：}
项目属于基础工程组件，聚焦渐进式发布、kill switch 和运行时配置决策。MoonBit 生态正在扩展到服务端、Wasm 与工程化教学场景，这些场景常需要按用户、环境、地区或百分比灰度开启功能。MoonFlagKit 提供一个小而可测试的内核，使上层示例或框架无需先实现完整配置平台，也能获得可解释的灰度发布能力。

\textbf{核心功能：}
\begin{itemize}
  \item 布尔功能开关：支持开启、关闭和默认关闭行为。
  \item 精准规则：支持 allowlist 与 denylist，且 denylist 优先级更高。
  \item 上下文条件：支持 environment 和自定义属性等值匹配。
  \item 百分比灰度：使用稳定 bucket，使同一用户在同一 salt 下结果稳定。
  \item 配置解析与诊断：支持行式配置，检查重复 flag、空名称和非法 rollout。
  \item 决策解释与 CLI 示例：每次评估返回原因，并提供可运行命令行演示。
\end{itemize}

\textbf{技术路线：}
项目把功能开关定义为 \texttt{Flag}，把运行时输入定义为 \texttt{EvalContext}。评估过程按固定顺序执行：整体关闭、denylist、allowlist、环境与属性条件、百分比灰度分桶。灰度分桶使用 \texttt{bucket(subject, salt)} 得到 0--99 的稳定整数，只要用户标识和 salt 不变，结果就不会漂移。解析器采用简洁的行式配置，后续可在不改变核心评估函数的前提下增加 JSON、TOML 或 OpenFeature 风格 provider。

\textbf{原创与不重合说明：}
本项目为原创实现，不移植已有开源项目源码，不引入外部 MoonBit 依赖。重新选题前已检索 Mooncakes 相关项目，发现原路由方向已有同类包，因此改为功能开关与灰度发布决策方向。当前源码、测试、文档和 CLI 示例均围绕 MoonFlagKit 重新组织，不复用旧路由项目的业务设计。

\textbf{当前完成情况与后续计划：}
仓库已包含 MoonBit 核心库、黑盒测试、CLI 示例、README、设计文档、验收清单和申报材料。测试覆盖功能关闭、白名单、黑名单、属性条件、稳定分桶、配置解析和诊断输出。后续计划支持多变体实验、JSON/TOML 配置入口、更丰富的条件表达式、OpenFeature 风格接口，并尝试与 MoonBit Web/Wasm 示例集成。
